## Périmètre fonctionnel

Application de bureau **locale** pour préparer, simuler, valider et exporter
les campagnes d’augmentation salariale.

### Couvert dans cette pré-recette

- Campagnes et configuration organisationnelle locale
- Import RH (format actuel)
- Référentiels de compensation
- **Simulations** de campagne (contrat de calcul v4 / snapshot schema v3)
- **Sauvegarde et historique** des runs (append-only)
- **Promotion** structurée (coûts et trajectoire)
- **Minimum garanti**
- **Incidence d’ancienneté**
- **Export Excel RH** (5 feuilles, dont tableau de bord)
- **Chiffrement par mot de passe** (Agile) et écriture atomique
- **Tableau de bord RH** (indicateurs, tranches de taux, graphiques)
- Exécution **100 % locale** : aucune transmission réseau métier

### Limites connues

- Personnalisation client (marque / Orange, etc.) **non encore réalisée**
- Licence annuelle **non encore intégrée**
- Signature numérique des installeurs **non encore intégrée**
- Aucun mécanisme d’expiration ni d’updater automatique
- Enrichissement éventuel de l’import RH reporté
- Version destinée à la validation métier, pas au déploiement définitif

## Installation (Windows)

1. Obtenir le package de pré-recette (`.msi` ou installeur NSIS `.exe`).
2. Exécuter l’installeur sur un poste Windows avec WebView2 Runtime disponible
   (ou autoriser son installation via le bootstrapper système).
3. Lancer **JRB Compensation Studio** depuis le menu Démarrer.
4. Vérifier l’ouverture de la fenêtre « JRB Compensation Studio ».

Packages cibles de cette livraison : **MSI** et **NSIS**.

## Désinstallation

1. Paramètres Windows → Applications → JRB Compensation Studio → Désinstaller  
   (ou via le panneau de configuration / l’installeur MSI/NSIS).
2. **Avant** désinstallation ou changement de poste : sauvegarder la base locale
   (voir ci-dessous). La désinstallation peut laisser ou supprimer des fichiers
   selon le mode d’installeur ; ne pas compter sur une conservation automatique
   des données métier.

## Emplacement local des données

Sous Windows, la base SQLite de l’application est typiquement créée sous :

`%APPDATA%\com.jrbxsolutions.compensationstudio\jrb-compensation-studio.db`

Exemple constaté en développement :

`C:\Users\<utilisateur>\AppData\Roaming\com.jrbxsolutions.compensationstudio\jrb-compensation-studio.db`

### Avertissement de sauvegarde

**Sauvegarder ce fichier (et toute copie de travail) avant toute désinstallation,
réinstallation ou changement de poste.** Les campagnes, imports, simulations et
exports ne sont pas synchronisés vers un serveur : une perte locale est
définitive.
