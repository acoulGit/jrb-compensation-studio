## Périmètre fonctionnel disponible sur la base Git

Application de bureau **locale** pour préparer, simuler, valider et exporter
les campagnes d’augmentation salariale.

### Couvert

- Campagnes et configuration organisationnelle locale
- Accès local (mot de passe) et période initiale
- Activation de **licence hors ligne** (renouvellement)
- Import RH (format actuel)
- Référentiels de compensation
- Simulations de campagne (contrat de calcul **v9** / snapshot schema **v7**)
- Migration SQLite courante : **0013** (`universal_fixed_amount`)
- Mécanismes sociaux exclusifs :
  - aucun mécanisme social ;
  - minimum garanti ;
  - forfait social universel
- Sauvegarde et historique des runs (append-only)
- Promotions structurées, incidence d’ancienneté, rétroactivité configurable
- Export Excel RH (chiffrement Agile optionnel)
- Exécution **100 % locale** : aucune transmission réseau métier

### Hors périmètre de cette prérecette

- Pages « Revue individuelle » et « Rapports » (placeholders, non destinées
  aux tests métier de cette livraison)
- Signature Authenticode des installateurs

## Formats Windows prévus

Packages cibles des blocs de packaging ultérieurs : **MSI** et **NSIS**.

- Signature Authenticode : **absente**
- Avertissement Windows SmartScreen / « éditeur inconnu » : **possible**
- Prérequis poste : Windows 10/11 x64, WebView2 Runtime

Les artefacts d’installation ne sont **pas** livrés avec ce seul alignement
de versionnement.

## Emplacement local des données

Sous Windows, la base SQLite est typiquement créée sous :

`%APPDATA%\com.jrbxsolutions.compensationstudio\jrb-compensation-studio.db`

### Sauvegarde / restauration

Actuellement **manuelle** : fermer l’application, copier le fichier `.db`
(et éventuels fichiers `-wal` / `-shm` si présents), puis restaurer par
remplacement contrôlé. Aucune synchronisation serveur.

**Sauvegarder ce fichier avant toute désinstallation, réinstallation ou
changement de poste.**
